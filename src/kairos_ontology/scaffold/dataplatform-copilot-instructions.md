## Project overview

This is a **dbt dataplatform project** that consumes medallion layer models
(silver/gold) generated by the linked ontology-hub repository. The ontology-hub
produces dbt packages via GitHub Releases; this repo pins to a specific version
and runs `dbt deps` to pull them in.

### Architecture

```
ontology-hub (producer)              this repo (consumer)
┌────────────────────────┐           ┌────────────────────────┐
│ model/ontologies/      │           │ dbt_project.yml        │
│ model/extensions/      │           │ packages.yml ──────┐   │
│ integration/mappings/  │           │ models/ (custom)   │   │
│                        │           │ dbt_packages/ ◄────┘   │
│ output/medallion/dbt/  │ ────────► │   └── ontology_hub/    │
└────────────────────────┘           └────────────────────────┘
         │ GitHub Release (tag)
         └── consumed via packages.yml
```

## Dev toolchain

- **[uv](https://docs.astral.sh/uv/)** manages the virtual environment and dependencies.
- Run `uv sync` to create/refresh the `.venv` and install all dependencies.
- Run toolkit commands with `uv run kairos-ontology <command>`.
- **dbt** is the core data transformation tool:
  - `dbt deps` — pull hub packages
  - `dbt build` — run models + tests
  - `dbt test` — run data tests only
- Profiles live in `~/.dbt/profiles.yml` (never committed to git).
- Use `profiles.yml.example` as a template for your local setup.

## dbt conventions

- Models follow the medallion architecture: **bronze** (raw) → **silver** (cleaned,
  typed, SCD) → **gold** (star schema, business metrics).
- Silver and gold models come from the ontology-hub package — do NOT edit them directly.
- Custom models go in `models/custom/` and may reference hub-generated models.
- Source bindings live in `models/_sources.yml` — update database/schema per environment.
- Macros go in `macros/`.
- Use `ref()` for internal models, `source()` for external tables.

## Package consumption workflow

1. The ontology-hub team releases a new version (e.g., `v1.4.0`).
2. Update `packages.yml` to pin the new version.
3. Run `dbt deps` to pull the updated models.
4. Run `dbt build` to verify compatibility.
5. If there are breaking changes, check the hub's CHANGELOG.
